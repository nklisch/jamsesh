playground-onboarding

The first-touch journey for prospective jamsesh users: arrive at the portal, install the CLI, spin up a real ephemeral session from a local checkout, share a join URL, watch collaborators join with auto-minted handles, and (eventually) see the session age out and destroy itself.

Hybrid topology. The primary path is sequential (1 → 7), but the share-link surface (step 4) is a modal cross-jump from the creator session (step 3), and the post-destruction page (step 7c) loops back to step 1 as a "try another" CTA. Step 7 is intentionally a single page stacking three sub-states because they're all variants on session-end, not independent moments.

How to read

  • step default-sequence card
  • ↩ from N reachable from a non-adjacent step (cross-jump)
  • ↻ loops to N end-of-flow loops back into the sequence
1 prospect-landing

Unauthenticated portal landing

Pitch + install one-liner + `jamsesh playground new` command. "I already have it installed" link skips straight to step 2.

↩ from 7c
2 create-cli

Terminal: after `jamsesh playground new`

The CLI's stdout — pushes local HEAD as base, mints the creator's nickname, prints the share URL and the expiry summary. "Open in browser? [Y/n]" prompt.

3 creator-session

Creator session view (anon mode)

SessionViewShell with playground chip, countdown badge (hard-cap + idle), no org name, anon handle in chrome. "You're alone — share the URL" nudge banner.

4 share-link

Share-link composer modal

Modal overlay on creator session view: copy URL, copy CLI one-liner, ephemeral reminder with concrete end-time. "Done" closes back to step 3.

↩ from 3
5 joiner-picker

Joiner nickname picker

Joiner clicks share URL: session card (host, goal, end-time) + nickname input pre-filled with server suggestion ("quiet-fox"), reroll button, customize-then-join.

6 joiner-session

Joiner session view

Same SessionViewShell, joiner perspective: host and joiner in presence, joiner's empty ref alongside host's commits, "welcome — bind via CLI" banner.

7 session-end

Session-end (3 sub-states)

7a idle-timeout warning (5 min before), 7b hard-cap warning (5 min before), 7c post-destruction page with stats + "try another" + "sign up for durable" CTAs.
